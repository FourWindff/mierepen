# 相似度搜索

相似度搜索是 RAG 检索环节的数学核心。理解它的原理，才能做出正确的技术选型。

## Embedding 的本质

Embedding 模型将文本映射到高维空间中的一个点。这个空间有一个关键性质：**语义相近的文本，在空间中距离更近。**

```
        高维向量空间（以 3D 示意）

              ↑ "机器学习"
             /|
            / |
    "AI" ←/  |  → "深度学习"
          |  |
          |  ↓ "猫"
          |
          └────────→ "狗"

"机器学习"和"深度学习"距离近（语义相关）
"猫"和"狗"距离近（都是动物）
"机器学习"和"猫"距离远（语义无关）
```

Embedding 向量的维度通常是 384~3072 维。维度越高，表达能力越强，但存储和计算开销也越大。

## 距离度量

### 欧氏距离（L2）

两点之间的直线距离，直觉上最直观：

```python
import numpy as np

def l2_distance(a: np.ndarray, b: np.ndarray) -> float:
    """欧氏距离。"""
    return float(np.linalg.norm(a - b))

# 等价写法
l2_dist = np.sqrt(np.sum((a - b) ** 2))
```

**特点**：对向量的绝对位置敏感，对大范数向量"惩罚"更重。

### 余弦相似度

两个向量夹角的余弦值：

```python
def cosine_similarity(a: np.ndarray, b: np.ndarray) -> float:
    """余弦相似度，范围 [-1, 1]。"""
    a_norm = a / np.linalg.norm(a)
    b_norm = b / np.linalg.norm(b)
    return float(np.dot(a_norm, b_norm))
```

**为什么 RAG 中余弦比欧氏更常用？**

Embedding 向量的**方向**编码语义，**长度**编码信息量。两个短文档和长文档可能说同一件事（方向相近），只是详细程度不同（长度不同）。余弦只关心方向，不关心长度，因此更适合语义匹配。

### 点积（Dot Product）

```python
def dot_product(a: np.ndarray, b: np.ndarray) -> float:
    return float(np.dot(a, b))
```

如果向量已经归一化（长度为1），点积 = 余弦相似度。很多向量库（如 FAISS 的 `IndexFlatIP`）使用点积，因为计算更快。

### 选择指南

| 度量 | 适用场景 | 注意 |
|------|----------|------|
| **余弦** | 通用语义检索 | 最常用，推荐默认选择 |
| **欧氏** | 需要区分信息量的场景 | 如图像特征匹配 |
| **点积** | 性能敏感、向量已归一化 | 等同于归一化后的余弦 |

## 精确最近邻搜索

暴力枚举所有向量，计算与查询向量的距离，取最小的 K 个：

```python
import numpy as np

def brute_force_knn(
    query: np.ndarray,
    vectors: np.ndarray,
    k: int = 5,
) -> tuple[np.ndarray, np.ndarray]:
    """
    暴力搜索 Top-K 最近邻。

    Args:
        query: 查询向量 (D,)
        vectors: 向量库 (N, D)
        k: 返回数量

    Returns:
        distances: (k,) 距离
        indices: (k,) 索引
    """
    # 计算与所有向量的距离
    # 假设向量已归一化，用点积 = 余弦
    similarities = vectors @ query  # (N,)

    # 取 top k（最大的相似度）
    top_k_idx = np.argpartition(similarities, -k)[-k:]
    top_k_idx = top_k_idx[np.argsort(similarities[top_k_idx])[::-1]]

    return similarities[top_k_idx], top_k_idx
```

**时间复杂度**：`O(N * D)` — 与向量库大小 N 成正比。

**适用**：N < 10K 的小规模场景。

## 近似最近邻（ANN）

当 N 达到百万甚至十亿级，暴力搜索太慢。ANN 算法牺牲少量精度（如召回率从 100% 降到 99%），换取数量级的速度提升。

### ANN 算法的核心思想

与其计算查询与**所有**向量的距离，不如先快速缩小搜索范围：

```
暴力搜索：          ANN 搜索：
┌──────────┐       ┌──────────┐
│ 所有向量  │       │ 粗筛范围  │ ← 快速排除大部分
│ ○ ○ ○ ○ │       │ ○ ○      │
│ ○ ○ ○ ○ │  →→→  │ ○ ○      │
│ ○ ○ ○ ○ │       └──────────┘
│ ○ ○ ○ ○ │            ↓
└──────────┘       ┌──────────┐
                   │ 精确搜索  │ ← 在小范围内精确计算
                   │ ○ ○      │
                   └──────────┘
```

### 常见 ANN 算法

| 算法 | 核心思想 | 优点 | 缺点 |
|------|----------|------|------|
| **HNSW** | 分层导航小世界图 | 高精度、速度快、支持增量 | 内存占用大 |
| **IVF** | 倒排文件：聚类后只搜最近簇 | 内存友好 | 需要训练、增量困难 |
| **PQ** | 乘积量化：压缩向量 | 存储极小 | 精度损失较大 |
| **LSH** | 局部敏感哈希 | 理论优雅 | 实际效果一般 |

生产环境最常用的是 **HNSW** 和 **IVF_PQ**（两者组合）。

## HNSW 深入理解

HNSW（Hierarchical Navigable Small World）是当前最流行的 ANN 算法，理解它的结构有助于调参。

### 分层图结构

```
Layer 2 (稀疏层):    ●─────●
                    /       \
                   ●─────────●

Layer 1 (中间层):    ●───●───●
                    / \ / \ / \
                   ●───●───●───●

Layer 0 (完整层):    ●─●─●─●─●─●─●
                    │╲│╱│╲│╱│╲│
                    ●─●─●─●─●─●─●
                    (所有节点 + 邻居连接)
```

- **Layer 0**：包含所有向量，连接最密集
- **上层**：节点的子集，连接更稀疏，用于快速"跳跃"
- **搜索**：从顶层随机节点开始，逐层向下贪婪导航到最近节点

### HNSW 关键参数

| 参数 | 含义 | 调参建议 |
|------|------|----------|
| `M` | 每个节点的最大邻居数 | 数据维度越高，`M` 越大。通常 16-64 |
| `efConstruction` | 建图时的搜索宽度 | 越大图质量越好，建图越慢。通常 100-400 |
| `efSearch` | 搜索时的搜索宽度 | 越大精度越高，搜索越慢。通常 50-400 |

```python
import faiss

d = 1536  # 向量维度
M = 32
ef_construction = 200

index = faiss.IndexHNSWFlat(d, M)
index.hnsw.efConstruction = ef_construction

# 添加数据
index.add(vectors)

# 搜索（可调精度-速度权衡）
index.hnsw.efSearch = 128  # 默认 = M，增大提高精度
D, I = index.search(query, k=10)
```

### 精度-召回权衡

```python
def benchmark_hnsw(index, queries, ground_truth, ef_search_values):
    """测试不同 efSearch 下的召回率。"""
    for ef in ef_search_values:
        index.hnsw.efSearch = ef
        _, indices = index.search(queries, k=10)

        # 计算召回率
        recall = np.mean([
            len(set(idx) & set(gt)) / len(gt)
            for idx, gt in zip(indices, ground_truth)
        ])

        print(f"efSearch={ef}: Recall@10={recall:.4f}")

# 典型输出：
# efSearch=16:  Recall@10=0.8234
# efSearch=64:  Recall@10=0.9512
# efSearch=128: Recall@10=0.9876
# efSearch=256: Recall@10=0.9967
```

根据业务对精度的要求，选择最小满足需求的 `efSearch`，以换取更快的速度。

## 评估指标

### Recall@K

Top-K 结果中包含多少"真正相关"的文档：

```python
def recall_at_k(retrieved_ids: list[int], relevant_ids: set[int], k: int) -> float:
    """召回率 = 检索到的相关文档 / 所有相关文档"""
    hits = len(set(retrieved_ids[:k]) & relevant_ids)
    return hits / len(relevant_ids)
```

### MRR（Mean Reciprocal Rank）

第一个相关结果的排名的倒数：

```python
def mrr(retrieved_ids: list[int], relevant_ids: set[int]) -> float:
    for rank, doc_id in enumerate(retrieved_ids, 1):
        if doc_id in relevant_ids:
            return 1.0 / rank
    return 0.0
```

### NDCG（Normalized Discounted Cumulative Gain）

考虑相关度**分级**（不是简单的 relevant/irrelevant）：

```python
def dcg(scores: list[float]) -> float:
    """DCG = sum(score_i / log2(i+2))"""
    return sum(s / np.log2(i + 2) for i, s in enumerate(scores))

def ndcg(retrieved_scores: list[float], ideal_scores: list[float]) -> float:
    """NDCG = DCG / IDCG"""
    actual_dcg = dcg(retrieved_scores)
    ideal_dcg = dcg(sorted(ideal_scores, reverse=True))
    return actual_dcg / ideal_dcg if ideal_dcg > 0 else 0.0
```
