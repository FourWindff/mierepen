# 核心算法

RAG 的效果不仅取决于工程实现，更取决于算法层面的选择和优化。本分组深入讲解 RAG 系统的核心算法原理，帮助你从"调参工程师"成长为"理解为什么这样调参"的工程师。

## 为什么需要理解算法

- **模型选择**：Embedding 模型的维度、训练方式如何影响检索效果？
- **参数调优**：HNSW 的 `efConstruction` 和 `M` 到底在控制什么？
- **效果瓶颈**：检索不准，是向量距离的问题还是索引结构的问题？
- **方案选型**：什么时候用暴力搜索，什么时候用近似索引？

## 本分组章节

1. **相似度搜索** — Embedding 空间的数学基础、距离度量与最近邻算法
2. **混合检索** — 向量检索 + 关键词检索的融合策略
3. **重排序** — 用更精确的模型对初筛结果进行二次排序
4. **高级索引** — HNSW、IVF 等近似索引的结构与权衡

</Callout>
