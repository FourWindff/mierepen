# 检索

检索是 RAG 的"咽喉"环节——它决定了模型能看到什么信息。再强的生成模型，也只能基于检索到的上下文来回答。本章深入检索的核心机制和优化策略。

## 相似度计算

### 余弦相似度

最常用的相似度度量。两个向量夹角的余弦值，范围 `[-1, 1]`：

```python
import numpy as np

def cosine_similarity(a: np.ndarray, b: np.ndarray) -> float:
    """计算两个向量的余弦相似度。"""
    a_norm = a / np.linalg.norm(a)
    b_norm = b / np.linalg.norm(b)
    return float(np.dot(a_norm, b_norm))
```

**为什么用余弦而非欧氏距离？**

Embedding 向量的方向编码语义，长度编码的是"信息量"而非"相似度"。两个内容相关但长度不同的文档，方向相近但欧氏距离可能很远。

### 点积 vs 余弦

| 度量 | 公式 | 特点 |
|------|------|------|
| **点积** | `a · b` | 同时考虑方向和大小的匹配 |
| **余弦** | `(a·b) / (||a||||b||)` | 仅考虑方向，对长度不敏感 |
| **欧氏距离** | `||a - b||` | 适合空间邻近性任务 |

Embedding API 通常返回归一化向量，此时点积 = 余弦相似度。

## 基础检索流程

```python
class VectorRetriever:
    def __init__(self, index, embedding_fn, chunks: list[DocumentChunk]):
        self.index = index
        self.embedding_fn = embedding_fn
        self.chunks = chunks

    def retrieve(self, query: str, top_k: int = 5) -> list[RetrievalResult]:
        # 1. 查询向量化
        query_embedding = self.embedding_fn(query)

        # 2. 向量搜索
        distances, indices = self.index.search(
            np.array(query_embedding).reshape(1, -1), top_k
        )

        # 3. 组装结果
        results = []
        for dist, idx in zip(distances[0], indices[0]):
            results.append(RetrievalResult(
                chunk=self.chunks[idx],
                score=float(dist),
                rank=len(results) + 1
            ))
        return results
```

## 检索常见问题与解决

### 问题 1：查询与文档表达不一致

用户问"怎么退款"，文档写的是"退货政策"。字面不同，但语义相近。

**解决方案：查询扩展（Query Expansion）**

```python
def expand_query(query: str, llm_client) -> list[str]:
    """用 LLM 生成查询的同义表达，多路检索后合并结果。"""
    prompt = f"""
    为以下查询生成 3 个不同的表达方式，保持语义不变：
    查询: {query}

    只用换行分隔，不要编号：
    """
    response = llm_client.complete(prompt)
    variations = [q.strip() for q in response.split('\n') if q.strip()]
    return [query] + variations

# 多路检索 + 去重 + 重排
all_results = []
for q in expand_query("怎么退款", llm):
    all_results.extend(retriever.retrieve(q, top_k=5))

# 按 chunk id 去重，保留最高相似度
from collections import defaultdict
best_scores = defaultdict(float)
for r in all_results:
    if r.score > best_scores[r.chunk.id]:
        best_scores[r.chunk.id] = r.score
```

### 问题 2：检索到无关内容

**解决方案：相似度阈值过滤**

```python
def retrieve_with_threshold(query: str, top_k: int = 5,
                            min_score: float = 0.7) -> list[RetrievalResult]:
    results = retriever.retrieve(query, top_k=top_k)
    return [r for r in results if r.score >= min_score]
    # 如果全部低于阈值，可以返回空结果 + 提示"知识库中没有相关信息"
```

### 问题 3：需要多跳推理

用户问："张三在公司负责的项目有哪些？"这需要先找到"张三的职位"，再找到"该职位负责的项目"。

**解决方案：迭代检索（Iterative Retrieval）**

```python
def iterative_retrieve(query: str, max_iterations: int = 3) -> list[DocumentChunk]:
    context = []
    current_query = query

    for _ in range(max_iterations):
        results = retriever.retrieve(current_query)
        new_chunks = [r.chunk for r in results if r.chunk not in context]
        if not new_chunks:
            break

        context.extend(new_chunks)

        # 用 LLM 判断是否需要进一步检索
        prompt = f"""
        原始问题: {query}
        已检索到的信息: {[c.content for c in context]}

        还需要检索什么额外信息才能完整回答问题？
        如果已有足够信息，回答 "ENOUGH"。
        否则给出新的检索查询：
        """
        next_query = llm.complete(prompt).strip()
        if next_query == "ENOUGH":
            break
        current_query = next_query

    return context
```

## 元数据过滤

纯向量检索无法处理结构化条件。结合元数据过滤可以大幅提升精度：

```python
# 示例：只检索最近一年的产品文档
results = collection.query(
    query_texts=["产品支持哪些格式?"],
    n_results=5,
    where={
        "$and": [
            {"doc_type": "product_manual"},
            {"created_at": {"$gte": "2025-01-01"}}
        ]
    }
)
```

## 检索评估

如何判断检索质量？

```python
from dataclasses import dataclass

@dataclass
class RetrievalMetrics:
    """检索质量指标"""
    recall_at_k: float      # Top-K 中 relevant 的比例
    mrr: float              # Mean Reciprocal Rank
    ndcg: float             # 归一化折损累积增益

def evaluate_retrieval(queries: list[EvalQuery], retriever) -> RetrievalMetrics:
    """
    queries: 每个查询标注了 relevant chunk ids
    """
    mrr_sum = 0
    recall_sum = 0

    for q in queries:
        results = retriever.retrieve(q.text, top_k=10)
        result_ids = [r.chunk.id for r in results]

        # MRR: 第一个 relevant 的排名倒数
        for rank, rid in enumerate(result_ids, 1):
            if rid in q.relevant_ids:
                mrr_sum += 1.0 / rank
                break

        # Recall@K
        hits = len(set(result_ids) & q.relevant_ids)
        recall_sum += hits / len(q.relevant_ids)

    n = len(queries)
    return RetrievalMetrics(
        recall_at_k=recall_sum / n,
        mrr=mrr_sum / n,
        ndcg=0.0  # 完整 NDCG 需要相关度分级标注
    )
```

## 实践建议

1. **始终设置阈值**：不要让低质量检索结果污染生成上下文
2. **查询扩展**：对关键词不明确的查询，用 LLM 生成多个表达变体
3. **混合检索**：向量检索 + 关键词检索（BM25）互补使用
4. **记录检索日志**：线上查询和检索结果是优化系统的宝贵数据
