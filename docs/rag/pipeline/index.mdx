# RAG 流程

本分组系统拆解 RAG 系统的完整流程。一个生产级的 RAG 系统不是简单的"向量检索 + LLM 调用"，而是由多个精密配合的环节组成。理解每个环节的作用和权衡，是构建可靠 RAG 系统的关键。

## 流程概览

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  数据准备     │───▶│  嵌入与索引   │───▶│    检索      │───▶│    生成      │
│  清洗/切分   │    │  向量化/存储  │    │  相似度搜索   │    │  提示工程    │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
```

## 本分组章节

1. **数据准备** — 原始数据的清洗、格式转换和分块策略
2. **嵌入与索引** — 选择 Embedding 模型、构建向量索引
3. **检索** — 相似度搜索的实现与优化
4. **生成** — 提示工程、上下文拼接与答案生成

<Callout type="warning" title="从原理出发">
本分组侧重原理和手工实现，帮助你理解每个环节"在做什么"。如果你想快速用现成框架搭建，可以跳过本分组直接看「LangChain 下的 RAG 流程」分组。
</Callout>
