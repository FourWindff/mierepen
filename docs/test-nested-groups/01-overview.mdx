# 概述

这是**测试分组教程**的顶层章节。它位于教程根目录下，不归属于任何分组。通过这个教程，你可以了解如何使用二级目录来组织内容。

## 为什么要使用分组

当教程内容较多时，将所有章节平铺在一个目录下会导致：

- 文件难以快速定位
- 章节之间的逻辑关系不清晰
- 侧边栏导航过长

引入二级目录（分组）后，可以将相关章节组织在一起：

<Callout type="info" title="分组的核心优势">
分组让教程结构更加清晰，读者可以快速找到感兴趣的内容，同时保持侧边栏的整洁。
</Callout>

## 本教程的结构

| 位置 | 说明 | 示例 |
|------|------|------|
| 根目录 | 顶层章节，不属于任何分组 | 本页"概述" |
| `basics/` | 基础概念分组 | 什么是 RAG、详细说明 |
| `advanced/` | 进阶技术分组 | 技术 A |

<Callout type="warning" title="注意">
顶层章节和分组可以在同一个教程中共存。顶层章节会显示在侧边栏的最前面，适合放置"概述"、"前言"这类内容。
</Callout>
